# API: Indexing Macros

### Scope

- **Purpose**: Define the API contract for all macros that wrap function definitions in named macros for lazy indexed expansion.
- **Responsibility**: Document operations, error handling, and compatibility for `impls1!`, `impls2!`, `impls3!`, `impls!`, `impls_optional!`, `tests_impls!`, `tests_impls_optional!`.
- **In Scope**: All function-wrapping and test-wrapping macros; the proc macro variant from `impls_index_meta`.
- **Out of Scope**: Index invocation (→ `api/002`), function definition utilities (→ `api/003`).

### Abstract

These macros accept one or more function definitions and wrap each in a named macro. The resulting named macros are lazy — the functions they contain are not emitted until invoked through `index!`. Each macro in this group differs in implementation strategy, rename support, strictness, and test-attribute behavior.

### Operations

Seven macros are available in this group:

- **`impls1!`** — Accepts one or more function definitions (free functions, any visibility). Wraps each in a named macro with unused-macro enforcement.
- **`impls2!`** — Accepts the same input as `impls1!`. Uses a different internal expansion mechanism. Functionally equivalent output to `impls1!`.
- **`impls3!` / `impls!`** — Accepts the same input. Produces macros that additionally support an `as NewName` argument at invocation time, enabling function renaming. `impls!` is a direct alias. Preferred for new code.
- **`impls_optional!`** — Identical to `impls1!` but relaxes unused-macro enforcement. Use for conditionally compiled functions that may not always be invoked.
- **`tests_impls!`** — Identical to `impls1!` but each wrapped function also receives the test attribute when expanded. Strict unused-macro enforcement applies.
- **`tests_impls_optional!`** — Identical to `impls_optional!` with additional test attribute injection on expansion.

All macros accept the standard function definition syntax: optional visibility modifier, function keyword, name identifier, parameter list, optional return type, function body.

### Error Handling

- Non-function items passed to any macro produce a compile error at the macro call site.
- Unused named macros generated by `impls1!`, `impls2!`, `impls3!`, `tests_impls!` trigger a compile error. Optional variants suppress this error.
- Duplicate function names within a single invocation produce a macro-redefinition compile error.
- Passing the `as NewName` rename argument to a macro generated by `impls1!` or `impls2!` produces a compile error — those macros do not accept arguments (only `impls3!`-generated macros support rename invocation).

### Compatibility Guarantees

- All macros are exported through the `exposed` and `prelude` namespaces.
- `impls1!` and `impls2!` are maintained for compatibility; new code should prefer `impls3!` / `impls!`.
- The `as NewName` rename argument at invocation time is only available for macros created by `impls3!` / `impls!`.
- `tests_impls!` and `tests_impls_optional!` inject the test attribute as part of the macro expansion, not at parse time.
